# Development Workflow

All new features follow this workflow.

---

## 1. Feature Request

A new feature is described by the user or product requirement.

Example:

"Users should be able to reply to emails from the inbox."

---

## 2. Architecture Planning

The Architecture Agent analyzes the request and creates an implementation plan.

The plan includes:

* backend changes
* frontend changes
* infrastructure changes
* security considerations

---

## 3. Approval

The implementation plan must be approved before coding starts.

---

## 4. Implementation

The feature is implemented by the following agents:

* Backend Agent
* Frontend Agent
* DevOps Agent

Each agent follows the existing architecture and coding patterns.

---

## 5. Validation

The Review Agent checks:

* architecture consistency
* code quality
* security
* integration

---

## 6. Documentation

After the feature is implemented, the documentation must be updated.

project.md should describe:

* the new feature
* APIs created
* architecture updates
